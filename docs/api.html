API Reference¶

Config¶

A Config object is required connect to Conservator. There are a variety of ways to create an instance of config:

  • Config File

  • Environment variables

  • Standard input

  • Manually using constructor

In general, use Config.default(), as this will try all of the above.

class FLIR.conservator.config.Config(email, key, url='https://flirconservator.com/')¶

Contains a user’s Email and API Key (token), to be used when authenticating operations on an instance of Conservator at a certain URL.

Parameters

  • email – The user’s email.

  • key – The user’s API key or token.

  • url – The URL of the conservator instance.

static default(save=True)¶

Gets the default config. This works by iterating through the various credential sources, and returning the first one that works. Sources are queried in this order:

  • Config file

  • Environment variables

  • User input

Parameters

save – If True, save the config for future use. This means a user won’t need to type them again.

static default_config_path()¶

By default, your config is saved in ~/.conservator_config.json.

static delete_saved_default_config()¶

Delete the default saved config, if it exists.

static from_config()¶

Creates a Config object from the JSON config file at the Config.default_config_path().

static from_environ()¶

Creates a Config object from environment variables.

The environment variables are CONSERVATOR_EMAIL, CONSERVATOR_API_KEY, and CONSERVATOR_URL.

static from_file(path)¶

Creates a Config object from a JSON config file.

Parameters

path – The path to the JSON config file.

static from_input()¶

Creates a Config object from standard input.

save_to_default_config()¶

Saves the Config to the Default config file, meaning this config will be loaded by Config.default().

save_to_file(path)¶

Saves the Config to as JSON.

This file can be loaded using Config.from_file().

Parameters

path – The file path to save to.

Conservator¶

class FLIR.conservator.conservator.Conservator(config)¶

Bases: FLIR.conservator.connection.ConservatorConnection

Conservator is the highest level class of this library. It will likely be the starting point for all queries and operations.

You can get an instance using the default Config::
>>> Conservator.default()
<Conservator at https://flirconservator.com>

You can create also create an instance by passing any Config object:

>>> config = Config(my_email, my_key, "https://localhost:3000")
>>> Conservator(config)
<Conservator at https://localhost:3000>
Parameters

config – The FLIR.conservator.config.Config object to use for this connection.

static default()¶

Returns a Conservator using FLIR.conservator.config.Config.default().

query(query, operation_base=None, include_fields=(), exclude_fields=(), **kwargs)¶

Provides an alternative way to prepare and run SGQLC operations.

Parameters

  • query – The SGQLC query to run.

  • operation_base – The base object of the query. Defaults to FLIR.conservator.generated.schema.Query.

  • include_fields – List of field paths to include in the returned object(s). Fields of subtypes can be specified as a path separated by .. For instance, when querying for a Dataset, you could add repository.master to only include the master field of Dataset.repository. If empty or not specified, all fields will be included.

  • exclude_fields – A list of field paths to exclude. These take the same form as included fields, and override them. In the above example, excluding repository will exclude all fields of repository.

  • kwargs – These named parameters are passed as arguments to the query.

static recursive_add_fields(obj, include_fields=(''), exclude_fields=(), path='', depth=0, max_depth=2)¶

Adds fields to an SGQLC operation.

Parameters

  • obj – The SGQLC object to add fields to.

  • include_fields – List of field paths to include in the returned object(s). Fields of subtypes can be specified as a path separated by .. For instance, when querying for a Dataset, you could add repository.master to only include the master field of Dataset.repository. If empty or not specified, all fields will be included.

  • exclude_fields – A list of field paths to exclude. These take the same form as included fields, and override them. In the above example, excluding repository will exclude all fields of repository.

  • path – The current path, used to filter included and excluded path fields.

  • depth – How many times this has been called recursively.

  • max_depth – The maximum depth of fields to add.

run(operation, variables=None)¶

Runs an SGQLC operation on the remote instance, and returns the response, if there were no errors.

If any errors are encountered, they will be raised with a ConservatorGraphQLServerError.

exception FLIR.conservator.connection.ConservatorGraphQLServerError(operation, errors)¶

There was a problem with a GraphQL query, and it’s unclear what the cause was.

Parameters

  • operation – The SGQLC operation that caused the error.

  • errors – A list of errors returned by the server.

exception FLIR.conservator.connection.ConservatorMalformedQueryException¶

There was a problem with a GraphQL query, and it’s the client’s fault.
